## ADDED Requirements

### Requirement: Create constant dialog access

The Configurator SHALL provide a "Create Constant" action in the editor toolbar area that opens a dialog for creating a new DHIS2 constant.

#### Scenario: User opens the create constant dialog

- **WHEN** the user clicks the "Create Constant" button in the Configurator toolbar
- **THEN** a dialog opens with fields for constant name and translations (en, es, fr, pt)

#### Scenario: Dialog is only available in configurator variant

- **WHEN** the app is running in the `autogenerated-forms` variant (form renderer)
- **THEN** no "Create Constant" button is visible

### Requirement: Constant name and code generation

The system SHALL auto-generate a constant code from the user-provided name by uppercasing, replacing non-alphanumeric characters with underscores, and applying the `AUTOGEN_` prefix.

#### Scenario: Code is derived from name

- **WHEN** the user enters "Section Header Label" as the constant name
- **THEN** the generated code SHALL be `AUTOGEN_SECTION_HEADER_LABEL`

#### Scenario: Special characters are sanitized

- **WHEN** the user enters "my-header (v2)" as the constant name
- **THEN** the generated code SHALL be `AUTOGEN_MY_HEADER_V2` (consecutive underscores collapsed)

#### Scenario: Name is required

- **WHEN** the user attempts to save a constant without entering a name
- **THEN** the dialog SHALL display a validation error and prevent saving

### Requirement: Translation entry during creation

The system SHALL allow the user to enter translations for supported locales (en, es, fr, pt) when creating a constant. Only the default description (en) is required.

#### Scenario: User provides translations

- **WHEN** the user enters a name, a description in English, and a description in French
- **THEN** the created constant SHALL include translations for `en` and `fr` locales on the `DESCRIPTION` property

#### Scenario: Optional translations left empty

- **WHEN** the user enters a name and English description but leaves es, fr, pt empty
- **THEN** the constant is created with only the English description and no additional translation entries

### Requirement: Constant persistence via DHIS2 API

The system SHALL persist the new constant to DHIS2 metadata using the existing `ConstantRepository`. The constant SHALL receive a valid DHIS2 UID.

#### Scenario: Successful creation

- **WHEN** the user fills in the name and at least the English description and clicks "Save"
- **THEN** the constant is created in DHIS2 with a generated UID, the derived code, and provided translations
- **AND** the dialog closes
- **AND** a success snackbar is shown with the constant code

#### Scenario: Duplicate code conflict

- **WHEN** the user tries to create a constant whose derived code matches an existing constant
- **THEN** the dialog SHALL display an error indicating the code already exists
- **AND** the constant is NOT created

#### Scenario: API failure

- **WHEN** the DHIS2 API returns an error during constant creation
- **THEN** the dialog SHALL display the error message
- **AND** the dialog remains open so the user can retry or cancel

### Requirement: Constant reference copied to clipboard

After successful constant creation, the system SHALL copy the JSON reference snippet (`{ "code": "<GENERATED_CODE>" }`) to the user's clipboard so they can paste it into the editor.

#### Scenario: Clipboard copy on success

- **WHEN** a constant is successfully created with code `AUTOGEN_MY_HEADER`
- **THEN** the string `{ "code": "AUTOGEN_MY_HEADER" }` is copied to the clipboard
- **AND** the success snackbar indicates the reference has been copied

#### Scenario: Clipboard unavailable

- **WHEN** a constant is successfully created but the clipboard API is not available
- **THEN** the success snackbar displays the code reference for manual copying
- **AND** no error is thrown
